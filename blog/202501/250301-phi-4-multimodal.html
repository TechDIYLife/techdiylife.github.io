1 简介

Phi-4-multimodal-instruct 是一个轻量级的开放多模态基础模型，利用了 Phi-3.5 和 Phi-4.0 版本的语言、视觉和语音研究成果及数据集。该模型能够处理文本、图像和音频输入，并生成文本输出，支持 128K token 的上下文长度。

模型经过了增强优化，包括 监督微调（Supervised Fine-Tuning）、直接偏好优化（Direct Preference Optimization）以及基于人类反馈的强化学习（RLHF, Reinforcement Learning from Human Feedback），以提高指令遵循能力和安全性。

各模态支持的语言如下：

  • 文本（Text）：阿拉伯语、中文、捷克语、丹麦语、荷兰语、英语、芬兰语、法语、德语、希伯来语、匈牙利语、意大利语、日语、韩语、挪威语、波兰语、葡萄牙语、俄语、西班牙语、瑞典语、泰语、土耳其语、乌克兰语
  • 视觉（Vision）：英语
  • 音频（Audio）：英语、中文、德语、法语、意大利语、日语、西班牙语、葡萄牙语

2 主要应用场景

该模型适用于广泛的 多语言、多模态 商业和研究用途，可用于通用人工智能系统和应用，特别是需要以下功能的场景：

  • 内存/计算受限环境
  • 低延迟场景
  • 强推理能力（特别是数学和逻辑推理）
  • 函数调用和工具调用
  • 通用图像理解
  • 光学字符识别（OCR）
  • 图表和表格理解
  • 多图像对比
  • 多图像或视频片段摘要
  • 语音识别
  • 语音翻译
  • 语音问答（Speech QA）
  • 语音摘要
  • 音频理解

3 版本说明

本次发布的 Phi-4-multimodal-instruct 模型基于 Phi-3 系列。

此前，用户需要使用语音识别模型与 Mini 和 Vision 模型交互，这通常涉及 两步模型流水线：

  1. 一个模型将音频转换为文本
  2. 另一个模型处理语言或视觉任务

这种方式意味着核心模型无法接收到完整的输入信息。例如，它无法直接感知多说话人、背景噪音，也无法在相同的表征空间中对语音、视觉和语言信息进行联合对齐。

Phi-4-multimodal-instruct 采用了一种新的开放多模态模型，能够同时处理文本、视觉和音频输入，所有输入和输出均由同一神经网络处理。

主要改进点

  • 新架构设计，提高多模态处理能力
  • 更大的词汇表，提升效率
  • 多语言 & 多模态支持，增强跨语言适用性
  • 改进的后训练技术，提升指令遵循能力和函数调用能力
  • 扩展数据规模，显著增强多模态能力

4 模型质量

为了评估能力，Phi-4-multimodal-instruct 在多个基准测试上与一系列模型进行了对比。

4.1 语音能力（Speech）

Phi-4-multimodal-instruct 在语音相关任务上的表现如下：

  • 强大的自动语音识别（ASR）和语音翻译（ST）能力，超越 业内领先的 ASR 模型 WhisperV3 以及 ST 模型 SeamlessM4T-v2-Large。
  • 在 Huggingface OpenASR 排行榜上排名第一，单词错误率（WER）仅 6.14%，相比 2025 年 1 月 17 日时排名第二的最佳模型（6.5%）更优。
  • 首个开源模型支持语音摘要（Speech Summarization），其性能接近 GPT-4o。
  • 在语音问答（Speech QA）任务上，与 Gemini-1.5-Flash 和 GPT-4o-realtime-preview 存在一定差距，未来版本将进一步优化此能力。

  • 整体表现：Phi-4-multimodal-instruct 平均优于 体量相当的竞争模型，并且在 多帧（multi-frame）能力上 能与更大规模的模型竞争。

5 训练信息（Training）

模型概述

  • 架构（Architecture）：Phi-4-multimodal-instruct 采用 5.6B 参数 的多模态 Transformer 模型，其语言模型基于 Phi-4-Mini-Instruct 预训练，并配备高级视觉和语音编码器及适配器。

  • 输入（Inputs）：支持 文本、图像和音频，最适合 对话格式（chat format） 的提示输入。

  • 上下文长度（Context Length）：128K tokens

  • 计算资源（GPUs）：512 张 A100-80G

  • 训练时长（Training Time）：28 天

  • 训练数据（Training Data）：

    • 5T tokens（文本数据）
    • 230 万小时语音数据（2.3M speech hours）
    • 1.1T 图像-文本对齐数据（1.1T image-text tokens）

  • 输出（Outputs）：根据输入生成文本

训练细节

  • 训练时间段：2024 年 12 月 – 2025 年 1 月
  • 数据截止日期：该模型为静态模型，仅基于离线数据训练，并且公开数据的截断日期为 2024 年 6 月。

支持的语言（Supported Languages）

  • 文本（Text）：阿拉伯语、中文、捷克语、丹麦语、荷兰语、英语、芬兰语、法语、德语、希伯来语、匈牙利语、意大利语、日语、韩语、挪威语、波兰语、葡萄牙语、俄语、西班牙语、瑞典语、泰语、土耳其语、乌克兰语
  • 视觉（Vision）：英语
  • 音频（Audio）：英语、中文、德语、法语、意大利语、日语、西班牙语、葡萄牙语

发布时间（Release Date）

2025 年 2 月

6 训练数据集（Training Datasets）

Phi-4-multimodal-instruct 的训练数据来源多样，总计包含 5 万亿（5T）文本 tokens，主要包括以下几类：

数据组成

  1. 高质量的公开数据、精选教育数据和代码（经过筛选过滤）
  2. 专门创建的合成“教科书式”数据，用于数学、编程、常识推理、通识知识（如科学、日常活动、心理理论等）的教学
  3. 高质量人工标注数据（对话格式）
  4. 精选高质量的图文混合数据（image-text interleave data）
  5. 合成及公开的图像、多图像和视频数据
  6. 匿名的内部语音-文本对齐数据（包含强/弱标注的转录文本）
  7. 精选的高质量公开语音数据和匿名内部语音数据（含任务特定的监督数据）
  8. 精选的合成语音数据
  9. 合成视觉-语音数据

数据筛选与处理

  • 重点关注数据质量，以提升模型的推理能力。
  • 对公开文档进行严格筛选，确保仅保留有助于推理的知识。例如：
    • 一场英超比赛当天的比赛结果可能是大型基础模型的良好训练数据，但对于小型模型而言，该信息会被移除，以节省参数容量，提高推理能力。
  • 数据收集过程：
    • 从公开可用文档中筛选信息，去除低质量或不必要的数据。
    • 过滤图像和文本数据，移除或清理可能包含个人信息的数据，以保护隐私。
  • 去污染（Decontamination）流程：
    • 对数据进行标准化和分词，然后生成 n-grams 并与基准数据集进行对比。
    • 匹配 n-grams 超过阈值的数据将被标记为污染数据并移除。
    • 生成详细的污染报告，总结匹配文本、匹配比例和筛选结果，供进一步分析。

import requests
import torch
import os
import io
from PIL import Image
import soundfile as sf
from transformers import AutoModelForCausalLM, AutoProcessor, GenerationConfig
from urllib.request import urlopen


# Define model path
model_path = "microsoft/Phi-4-multimodal-instruct"

# Load model and processor
processor = AutoProcessor.from_pretrained(model_path, trust_remote_code=True)
model = AutoModelForCausalLM.from_pretrained(
    model_path, 
    device_map="cuda", 
    torch_dtype="auto", 
    trust_remote_code=True, 
    attn_implementation='flash_attention_2',
).cuda()


# Load generation config
generation_config = GenerationConfig.from_pretrained(model_path)

# Define prompt structure
user_prompt = '<|user|>'
assistant_prompt = '<|assistant|>'
prompt_suffix = '<|end|>'


# Image Processing
print("\n--- IMAGE PROCESSING ---")
#image_url = 'https://www.ilankelman.org/stopsigns/australia.jpg'
image_url = 'https://azure.microsoft.com/en-us/blog/wp-content/uploads/2025/02/Azure_638390_Blog_250221.webp'
prompt = f'{user_prompt}<|image_1|>What is shown in this image? {prompt_suffix}{assistant_prompt}'

print(f'>>> Prompt\n{prompt}')

# Download and open image
image = Image.open(requests.get(image_url, stream=True).raw)
inputs = processor(text=prompt, images=image, return_tensors='pt').to('cuda:0')

# Generate response
generate_ids = model.generate(
    **inputs,
    max_new_tokens=1000,
    generation_config=generation_config,
)
generate_ids = generate_ids[:, inputs['input_ids'].shape[1]:]
response = processor.batch_decode(
    generate_ids, skip_special_tokens=True, clean_up_tokenization_spaces=False
)[0]
print(f'>>> Response\n{response}')



# Audio Processing
print("\n--- AUDIO PROCESSING ---")
audio_url = "https://upload.wikimedia.org/wikipedia/commons/b/b0/Barbara_Sahakian_BBC_Radio4_The_Life_Scientific_29_May_2012_b01j5j24.flac"
speech_prompt = "Transcribe the audio to text, and then translate the audio to Chinese. Use <sep> as a separator between the original transcript and the translation."
prompt = f'{user_prompt}<|audio_1|>{speech_prompt}{prompt_suffix}{assistant_prompt}'
print(f'>>> Prompt\n{prompt}')

# Downlowd and open audio file
audio, samplerate = sf.read(io.BytesIO(urlopen(audio_url).read()))

# Process with the model
inputs = processor(text=prompt, audios=[(audio, samplerate)], return_tensors='pt').to('cuda:0')

generate_ids = model.generate(
    **inputs,
    max_new_tokens=1000,
    generation_config=generation_config,
)
generate_ids = generate_ids[:, inputs['input_ids'].shape[1]:]
response = processor.batch_decode(
    generate_ids, skip_special_tokens=True, clean_up_tokenization_spaces=False
)[0]
print(f'>>> Response\n{response}')
