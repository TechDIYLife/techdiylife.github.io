10个问题快速了解 Gradio

1 什么是 Gradio？

Gradio 是一个开源的 Python 包，用于快速构建机器学习模型的演示或 Web 应用程序。 一旦构建好 Gradio 应用，就可以利用 Gradio 提供的功能轻松地共享应用。
使用 Gradio 制作Web应用程序完全不需要后端或前端的工程知识。

2 如何进行安装？

Gradio是 python 开发库，所以需要提前准备 python 运行环境。
然后只需要执行下面的命令， 即可完成 Gradio 的安装。

pip install --upgrade gradio

3 第一个 Gradio 演示程序

新建一个python脚本：gradio_sample1.py

import gradio as gr

def greet(name, intensity):
    return "Hello, " + name + "!" * int(intensity)

demo = gr.Interface(
    fn=greet,
    inputs=["text", "slider"],
    outputs=["text"],
)

demo.launch()

这个代码中，最主要的部分就是 gr.Interface, 我们可以看到，Interface 类在初始化时需要三个必填参数：

  • fn：与UI相对应的函数
  • inputs：用于输入的 Gradio 组件。组件数量应与函数的参数数量一致。
  • outputs：用于输出的 Gradio 组件。组件数量应与函数的返回值数量一致。

通过 fn，无论是简单的计算处理，还是图像生成模型等机器学习模型的运行，都可以用 UI 来封装任意的 Python 函数，这是一种非常灵活的设计。 在 inputs 和 outputs 参数中，可以指定一个或多个 Gradio 组件。Gradio 提供了 30 多种为机器学习应用设计的内置组件，例如 gr.Textbox()、gr.Image()、gr.Dropdown() 等。

4 如何启动与访问Gradio演示程序（三种访问方式）？

代码准备完成之后，可以通过下面的命令来启动服务：

python gradio_sample1.py

基于 Gradio 的程序，有三种不同的访问方式：

  • 情况1：在同一台机器上访问
    启动代码：demo.launch(server_port=7860)
    http://127.0.0.1:7860

  • 情况2：在同一个局域网中的机器访问
    启动代码：demo.launch(server_name="0.0.0.0", server_port=7860)
    访问URL： http://ip-address:7860

  • 情况3：在公共网络访问
    启动代码：demo.launch(share=True, server_port=7860)
    访问URL： （请查看启动时的提示）

5 启动失败：网络代理的影响？

如果碰到了以下错误，可能是你的网络代理的影响：

* Running on local URL:  http://127.0.0.1:7860
Traceback (most recent call last):
  File "xxxx.py", line 13, in <module>
    demo.launch()
  File "xxxx\Roaming\Python\Python311\site-packages\gradio\blocks.py", line 2819, in launch
    raise Exception(
Exception: Couldn't start the app because 'http://127.0.0.1:7860/gradio_api/startup-events' failed (code 503). Check your network or proxy settings to ensure localhost is accessible.

建议在程序中加入：

# 下面的代码要在 import gradio 之前
import os
os.environ['NO_PROXY'] = '127.0.0.1,localhost'

如果你碰到下面的错误，表示你的网络代理限制了网络穿透，无法设置公共网络的访问。

[W] [service.go:132] login to server failed: DialTcpByHttpProxy error, StatusCode [403]
Could not create share link. Please check your internet connection or our status page: https://status.gradio.app.

7 可以固定 Gradio 提供的 public url 吗？

在使用 share=True 启动服务是， public URL：xxxx.gradio.live
每次启动都会变化， 这个 URL 可以固定吗？

答案是不可以，每次生成的public URL都是不同的。

9 Gradio 提供哪些 GUI Interface？

  • Interface
    Interface 是 Gradio 的主要高层类，它可以让你在几行代码内，围绕一个机器学习模型（或任意 Python 函数）创建一个基于 Web 的 GUI/演示。在使用时，你必须指定三个参数：

    1. 需要创建 GUI 的函数
    2. 期望的输入组件
    3. 期望的输出组件

  • ChatInterface
    ChatInterface 是 Gradio 提供的用于创建聊天机器人 UI 的高层抽象类，它可以让你在几行代码内，围绕一个聊天机器人模型创建基于 Web 的演示。它只需要一个必填参数：fn。该参数接收一个函数，用于根据用户输入和聊天历史来生成聊天机器人的回复。

  • TabInterface
    TabbedInterface 是通过提供一个 Interface 或 Blocks 的列表来创建的，其中每一个都会在单独的标签页中渲染。
    在标签页中，只会渲染该 Interface/Blocks 中的组件。
    需要注意的是，某些 Blocks 的高层属性（例如自定义的 css、js 和 head 属性）将不会被加载。

10 Gradio 支持哪些组件？

Gradio 提供了30多种预构建的组件，只需一行代码即可作为 输入 或 输出 用于 Interface 或 Blocks。

这些组件包含预处理步骤：将用户通过浏览器提交的数据转换为 Python 函数可用的格式；
以及后处理步骤：将 Python 函数返回的值转换为浏览器可显示的内容。

更多信息，请参考：https://www.gradio.app/docs/gradio/introduction

11 GUI进阶：如何进一步自定义 GUI（Blocks组件）？

Blocks 是 Gradio 的底层 API，它允许你创建比 Interface 更加自定义的 Web 应用和演示（但依然完全使用 Python 实现）。

与 Interface 类相比，Blocks 提供了更多的灵活性和控制能力，主要体现在：

  1. 组件的布局
  2. 触发函数执行的事件
  3. 数据流（例如：输入可以触发输出，而输出又可以触发下一层的输出）

此外，Blocks 还支持将相关演示组合在一起，例如通过 tabs（标签页） 的形式。

Blocks 的基本用法如下：

  1. 创建一个 Blocks 对象
  2. 将其作为上下文（使用 with 语句）
  3. 在 Blocks 上下文中定义布局、组件或事件
  4. 最后调用 launch() 方法来启动演示

通过使用 Blocks 你可以设计出很复杂且美观的GUI，比如：

12 Flag 按钮是个什么鬼？ 你点过它吗？

你可能已经注意到，在 Interface 中默认会出现一个 “Flag” 按钮。
当用户在使用你的演示时，当它遇到一些问题，或者希望反馈问题时，可以点击该按钮，将对应的输入，输出记录下来，供你后续查看。

  • 内容会保存在哪里？
    默认情况下，会保存在 .gradio/flagged/ 目录下。
    你也可以在 Interface 构造函数中通过 flagging_dir= 参数指定存储的目录。

用户点击Flag按钮后，会生成一个 CSV 文件，来存储相关的记录。
如果界面涉及文件数据（例如 Image 或 Audio 组件），还会在该目录下创建文件夹，用于存储这些被标记的数据。
https://www.gradio.app/guides/flagging

14 我的demo运行很耗费资源，如何快速展示Demo？

可以通过 缓存示例（Caching examples）功能来实现。

当 cache_examples=True 时，在调用 launch() 方法时，Gradio 应用会运行所有示例并保存输出。
这些数据默认保存在工作目录下名为 gradio_cached_examples 的文件夹中。

你也可以将 cache_examples 设置为 "lazy"。这意味着每个示例只会在第一次被某个用户使用时才会被缓存。
如果你的预测函数运行时间较长，又不希望在启动 Gradio 应用时等待过久，这种方式会很有帮助。

需要注意的是，一旦缓存生成，在之后的启动中它不会自动更新。如果你更改了示例内容或函数逻辑，需要手动删除缓存文件夹，然后重新运行 launch() 方法以重建缓存。
