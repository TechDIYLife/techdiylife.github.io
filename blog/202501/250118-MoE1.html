2 混合专家模型：直觉 vs 实际

  • 直觉上的混合专家模型
    多个模型：专家模型1， 专家模型2， 专家模型3
    分别训练：数学专家模型，编程专家模型，翻译专家模型
    专家混合训练：将多个模型合并训练

  • 实际的混合专家模型
    单个模型：与标准LLM差异不大，只将部分层替换为了专家层
    单次训练：训练方法与传统模型一样，只训练一次

3 Mixtral是如何实现的？

Mixtral模型采用谷歌提出的 Switch Transformers 的方案，
参考论文：https://arxiv.org/pdf/2101.03961

  • Switch transformers 网络结构

  • Switch transformers auxiliary loss
    主要目的为了让MoE模型中的各个expert有均衡的负载（防止部分expert过载，而其他expert闲置）

    N: expert的数量
    α\alphaα 是一个超参数，用来调整auxiliary loss与token预测loss的的平衡

    fif_ifi​ 表示被分配到expert expertiexpert_iexperti​ 的token的比例，
    PiP_iPi​ 表示token被分配到 expertiexpert_iexperti​ 的概率。

    这是一个希望token能够均匀分配到各个expert的loss，当fif_ifi​ 和PiP_iPi​ 均等时，路由分配也会均等。

4 MoE模型实现

要实现MoE模型，需要在LLM模型中添加以下部分

  • expert layer： 定义个专家网络结构，一个或者多个层
  • Gate layer： 控制和选择expert的门控层
  • 专家部分的处理：对应上面的网络结构图
  • 辅助loss部分：控制多个专家的负载，防止过多选择专家，以及偏重部分专家的问题。

  • Code 3: MoE核心处理代码
class MixtralSparseMoeBlock(nn.Module):
    def __init__(self, config):
        # gating
        self.gate = nn.Linear(self.hidden_dim, self.num_experts, bias=False)
        # experts
        self.experts = nn.ModuleList([MixtralBlockSparseTop2MLP(config) for _ in range(self.num_experts)])
        # Jitter parameters
        self.jitter_noise = config.router_jitter_noise
    def forward(self, hidden_states: torch.Tensor) -> torch.Tensor:
        router_logits = self.gate(hidden_states)  # 计算每个 token 对各个 experts 的重要性分数（路由分数）
        routing_weights = F.softmax(router_logits, dim=1, dtype=torch.float) 
        routing_weights, selected_experts = torch.topk(routing_weights, self.top_k, dim=-1)  # 为每个 token 选择 top_k 个最相关的 experts 及其权重

        routing_weights /= routing_weights.sum(dim=-1, keepdim=True)  # 对选中的 weights 进行归一化，以便后续加权计算

        # 初始化输出张量，用于存储最终的 hidden states
        final_hidden_states = torch.zeros(
            (batch_size * sequence_length, hidden_dim), dtype=hidden_states.dtype, device=hidden_states.device
        )

        # 将选中的 experts 转换为 one-hot 编码的 mask，用于标记每个 token 所分配的 experts
        expert_mask = torch.nn.functional.one_hot(selected_experts, num_classes=self.num_experts).permute(2, 1, 0)  # shape: (num_experts, top_k, batch_size * sequence_length)

        # 遍历所有 experts，处理分配到的 token
        for expert_idx in range(self.num_experts):
            expert_layer = self.experts[expert_idx]  # 获取当前 expert 层
            idx, top_x = torch.where(expert_mask[expert_idx])  # 找出当前 expert 负责处理的 token 索引
            
            # 提取分配给当前 expert 的 token 的 hidden states，并调整形状以适配 expert 的输入格式
            current_state = hidden_states[None, top_x].reshape(-1, hidden_dim)  # shape: (num_tokens, hidden_dim)
            
            # 计算当前 expert 的输出，并乘以路由权重（routing_weights），以融合路由概率
            current_hidden_states = expert_layer(current_state) * routing_weights[top_x, idx, None]
            
            # 将当前 expert 的输出累加到 final_hidden_states 对应位置
            final_hidden_states.index_add_(0, top_x, current_hidden_states.to(hidden_states.dtype))

        # 恢复最终输出的形状，与输入保持一致
        final_hidden_states = final_hidden_states.reshape(batch_size, sequence_length, hidden_dim)
        return final_hidden_states, router_logits

class MixtralBlockSparseTop2MLP(nn.Module):
    def __init__(self, config: MixtralConfig):
        super().__init__()
        self.ffn_dim = config.intermediate_size
        self.hidden_dim = config.hidden_size

        self.w1 = nn.Linear(self.hidden_dim, self.ffn_dim, bias=False)
        self.w2 = nn.Linear(self.ffn_dim, self.hidden_dim, bias=False)
        self.w3 = nn.Linear(self.hidden_dim, self.ffn_dim, bias=False)

        self.act_fn = ACT2FN[config.hidden_act]

    def forward(self, hidden_states):
        current_hidden_states = self.act_fn(self.w1(hidden_states)) * self.w3(hidden_states)
        current_hidden_states = self.w2(current_hidden_states)
        return current_hidden_states

6 模型初始化与训练

  • α\alphaα 超参数的设置
    Qwen2-MoE代码默认值：router_aux_loss_coef=0.001
    Switch Transformers中使用的是 0.01

    论文中测试0.1到0.00001，最终选择 0.01，其值足够大以确保负载平衡，同时又足够小，不会压制主要的交叉熵目标。
    具体请参考下面论文的p8：https://arxiv.org/pdf/2101.03961
